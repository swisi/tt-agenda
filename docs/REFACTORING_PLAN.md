# Refactoring-Plan (v2)

## Phase 1: Foundations

- Zielarchitektur final festlegen
- Minimalen App-Skeleton aufbauen
- Konfigurationsmodell vereinheitlichen

## Phase 2: Core Migration

- Domain-Modelle aus Legacy analysieren
- Use Cases extrahieren und neu implementieren
- Persistenzadapter schrittweise anbinden

## Phase 3: Interface Migration

- HTTP-Routen auf neue Use Cases umstellen
- Templates/Frontend schrittweise migrieren
- Legacy-Pfade entfernen

## Phase 4: Stabilisierung

- Testabdeckung erhöhen
- Performance-/Security-Review
- Betriebsdoku finalisieren

## Definition of Done

- Feature-Parität für Kernfunktionen
- Alle kritischen Tests grün
- Doku und Runbooks vollständig
